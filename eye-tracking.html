Eye Tracking Research

Summary

Project Directive

Conduct usability testing while recording eye-tracking data to:

  1. Identify the most salient UI elements.
  2. Analyze how users interact with content.

My Contributions

As the lead UX designer I on the project I:

  1. Designed research protocols for test sessions.
  2. Configured eye tracking hardware.
  3. Moderated testing sessions with users.
  4. Analyzed eye tracking data.
  5. Analyzed subjective user feedback.

Eye Tracking Data

Reading Pattern

Conclusions:

  • Attention dissapates as users progress down the page.
  • Users focus on paragraph content, rather than UI elements like tags, share links, and call outs.

Gaze Pattern

Conclusions:

  • On search pages, tags and filters receive a great deal of attention.
  • Users pay little attention to search results further down the page.
Eye Tracking Data

Salient UI Elements

Conclusions:

  • The preferred method to navigatet to specific content is with tags, not FAQs.
  • "Other Categories" should be broken into additional sections for clearer IA.

Scanning Search Results

Conclusions:

  • The few few search results receive the bulk of attention.
  • Users pay partial attention to left-margin filters. They conduct a new search rather than interact with the filters.
